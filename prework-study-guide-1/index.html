Prework Study Guide

HTML

  • Starting off any code with '!DOCTYPE html' prepares the computer to render it in said coding language.
  • The 'doctype' command is not case sensitive.
  • The 'head' element contains the information about the webpage.
  • The 'body' element represents the visible content shown to the user.
  • The 'title' element is used for the title of the webpage.
  • Adding 'html lang="eng"' lets the browser know the website is written in english.
  • The 'p' element is for paragraphs.
  • The 'br' element is for breaks between elements in the page.
  • The 'ul' element is for unordered lists, but the 'ol' element is for ordered lists (AKA: numbered list).
  • Both types of lists need the 'li' element to add points to the list.
  • An image 'src' allows the computer to locate which files an image is located in.
  • The 'a' (anchor) element starts off a link and href will allow the computer to find the link stated.
  • 'h1' should only be used once on a page.
  • The higher the header 'h' number is, the smaller the header size.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Inline CSS: Quick but hard to change in bulk. It can only apply to one element, but it sticks well once it's there.
  • Internal CSS: Uses the 'style' element in the 'head' element to set a rule. Can apply to an element in bulk, but also tedious to change.
  • External CSS: Using the 'link' element, a file can be linked into the main HTML 'head' element in order to place CSS over everything in a webpage.
  • CSS Rule: div {color: blue; text-align: justify;}
    - 'Div': Selector; The element the following CSS will apply to.
    - 'Color'/'text-align': Property; Further specifying what the following CSS will do within the element.
    - 'Blue'/'Justify': Value; The effect being done to the content within the element.
    - 'Color: blue;': Declaration; A prompt to change something.
    - '{color: blue; text-align: justify;}': Declaration block; A set of prompts to change one element.
  • CSS can be applied to multiple elements through the use of a comma (,) between the selectors before the curly braces ({}).
  • CSS can be applied to everything with the use of an asterick (*) in the first space.

Git

  • Start every git command with 'git'.
  • 'git status': checks what branch we are currently on.
  • 'git checkout -b branch-name': creates a new branch and switches to it.
  • 'git checkout branch-name or main': can be used to move from other branches to main. Can also be used from main to pre-exiting branches.
  • 'git branch -d branch-name': deletes a local branch.
  • 'git add': adds modifications from the current working branch and into the staging area.
  • 'git add -A': indicates that we want to add all changes. Everything together informs Git that it will all be added to the next commit.
  • 'git commit -m "commit message": commits the changes into the branch. The quoted line is to describe what the commit contains.
  • 'git pull origin main': Pulls the branch's modifications into a local environment. The use of 'origin' shows that it will be pulled from the GitHub repo. As for the 'main', it's pulling from the main branch in the repo.
  • 'git push origin branch-name': Pushes the changes from local branch and into the remote branch. The push command can both add changes to an existing branch or make a new remote branch.
